As letras próximas aos nomes dos arquivos no explorador do VS Code são indicadores de estado do controle de versão Git, como U (Untracked - arquivo não monitorado), M (Modified - arquivo modificado) e A (Added - arquivo adicionado ao controle de versão), que aparecem ao lado dos nomes dos arquivos para informar sobre suas alterações e estado no repositório. 
Entendendo os indicadores Git:
U (Untracked): Indica que o arquivo foi editado, mas ainda não foi adicionado ao controle de versão (Git).
M (Modified): Apresenta arquivos que foram modificados e cujas alterações já foram rastreadas e registradas pelo Git.
A (Added): Mostra arquivos que foram adicionados ao controle de versão através do git add, mas que ainda não foram commitados. 
Por que esses indicadores aparecem?
Esses indicadores são uma integração direta do VS Code com o sistema de controle de versão Git, que ajuda a gerenciar as mudanças nos seus arquivos de código. 
Como os indicadores são exibidos?
Eles aparecem ao lado dos nomes dos arquivos na barra lateral do explorador de arquivos (o explorador de arquivos é a barra que fica ao lado esquerdo do VS Code
